# Menu3（Finder 菜单增强）

macOS Finder 上下文菜单增强工具，支持触摸板手势触发和文件快捷操作。

## 功能特性

- 🖱️ **多种触发方式**：中键点击、三指轻点、三指按下、Option 组合
- 📁 **文件操作**：复制/移动文件、复制文件名/路径、快速新建文本文件
- 📁 **文件操作**：复制/移动文件、复制文件名/路径、快速新建文件夹/文本文件、前往文件夹
- 🔄 **智能粘贴**：从剪贴板保存文件/文本到当前目录
- 🚀 **应用快速打开**：保存最近应用历史，快速在当前目录打开应用
- ☁️ **全目录支持**：本地、iCloud、移动存储等所有位置可用
- 🔌 **开机启动状态**：菜单栏实时显示是否已设置开机启动，可一键引导开启
- 💳 **授权策略**：支持 90 天免费试用与 LemonSqueezy $9.9 终身买断

## 安装

### 方式 1：DMG 安装（推荐）
1. 下载 `Menu3-1.0.1.dmg`
2. 打开 DMG 文件
3. 拖动 Menu3 到 Applications 文件夹
4. 打开应用并授予 Accessibility 权限

### 方式 2：源码编译
```bash
git clone git@github.com:hisunwei/menu3.git
cd menu3
xcodegen generate
xcodebuild -project menu3.xcodeproj -scheme menu3 -configuration Release build
```

## 触发方式配置

在 Menu3 设置中配置：

- **中键点击**：外接鼠标中键点击（默认启用）
- **👆 三指轻点**：触摸板三指快速轻点（推荐触摸板）
- **👇 三指按下**：触摸板三指用力按下
- **⌥👆 Option + 三指轻点**：按住 Option 键同时三指轻点
- **⌥👇 Option + 三指按下**：按住 Option 键同时三指按下

## 快速验收用例（F1-F6）

1. **F1 新建文本文件**（空白区域触发菜单）
   - 在 `TargetFolder` 空白处触发菜单，点 `新建文本文件`
   - 期望：生成 `未命名.txt`（重名则 `未命名 2.txt`），并由文本编辑打开

2. **F2 单文件复制名称/路径**
   - 单选任意文件触发菜单，点 `复制文件名`、`复制文件路径`
   - 期望：`pbpaste` 分别返回文件名、绝对路径

3. **F3 多文件复制名称/路径**
   - 多选 2+ 文件触发菜单，点 `复制所有文件名`、`复制所有文件路径`
   - 期望：`pbpaste` 为多行输出，每行一个名称/路径

4. **F4 在此打开应用**
   - 触发菜单 → `在此打开应用 ▸ 选择应用…`
   - 期望：应用以当前目录上下文启动；再次触发时最近使用应用最多显示 5 个且去重

5. **F5 保存文件 来自粘贴板**
   - Finder 打开 `ClipboardSource`，全选复制（Cmd+A, Cmd+C）
   - 回到 `TargetFolder` 空白处触发菜单
   - 期望：出现 `保存文件 来自粘贴板`，点击后文件复制到当前目录，重名自动编号
   - 期望：保存后 Finder 自动选中新文件并进入重命名状态

6. **F6 保存成文本文件**
   - 先执行：`echo '粘贴板文本测试' | pbcopy`
   - 在 `TargetFolder` 空白处触发菜单
   - 期望：出现 `保存成文本文件`，点击后生成文本文件（重名自动编号）
   - 期望：保存后 Finder 自动选中新文件并进入重命名状态

## 复制/移动文件功能

1. 选中文件 → 触发菜单 → 点 `复制选中项` 或 `移动选中项`
2. 导航到目标目录 → 再次触发菜单 → 点 `复制 xxx 到这里` 或 `移动 xxx 到这里`
3. 确认对话框中的操作描述
4. 操作完成，临时缓存自动清除

## 技术细节

- **架构**：基于 macOS Accessibility API 的菜单栏常驻应用
- **触摸板检测**：MultitouchSupport.framework 私有 API
- **Finder 通信**：AppleScript 与 Finder 交互
- **系统支持**：macOS 12.0+
- **权限**：Accessibility（需要用户授予）

## 边界与已知问题

- 剪贴板无文件时，不显示 `保存文件 来自粘贴板`
- 剪贴板无文本时，不显示 `保存成文本文件`
- 同时有文件和文本时，两个菜单项都显示
- 重名处理正确：`未命名.txt` -> `未命名 2.txt`…
- 某些特殊权限的目录可能无法操作，查看 Console 日志获取详情
