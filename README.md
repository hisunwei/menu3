# RightMenu（Finder 右键菜单增强）

## 快速测试准备

先生成测试数据：

```bash
bash Scripts/prepare-test-data.sh
```

默认会创建：

- `~/Desktop/RightMenu-Test/ClipboardSource`
- `~/Desktop/RightMenu-Test/TargetFolder`

## 快速验收用例（F1-F6）

1. F1 新建文本文件（空白区域右键）
   - 在 `TargetFolder` 空白处右键，点 `新建文本文件`
   - 期望：生成 `未命名.txt`（重名则 `未命名 2.txt`），并由文本编辑打开

2. F2 单文件复制名称/路径
   - 单选任意文件右键，点 `复制文件名`、`复制文件路径`
   - 期望：`pbpaste` 分别返回文件名、绝对路径

3. F3 多文件复制名称/路径
   - 多选 2+ 文件右键，点 `复制所有文件名`、`复制所有文件路径`
   - 期望：`pbpaste` 为多行输出，每行一个名称/路径

4. F4 在此打开应用
   - 右键 `在此打开应用 ▸ 选择应用…`
   - 期望：应用以当前目录上下文启动；再次右键时最近使用应用最多显示 5 个且去重

5. F5 保存文件 来自粘贴板
   - Finder 打开 `ClipboardSource`，全选复制（Cmd+A, Cmd+C）
   - 回到 `TargetFolder` 空白处右键
   - 期望：出现 `保存文件 来自粘贴板`，点击后文件复制到当前目录，重名自动编号
   - 期望：保存后 Finder 自动选中新文件并进入重命名状态

6. F6 保存成文本文件
   - 先执行：`echo '粘贴板文本测试' | pbcopy`
   - 在 `TargetFolder` 空白处右键
   - 期望：出现 `保存成文本文件`，点击后生成文本文件（重名自动编号）
   - 期望：保存后 Finder 自动选中新文件并进入重命名状态

## 边界与回归检查

- 剪贴板无文件时，不显示 `保存文件 来自粘贴板`
- 剪贴板无文本时，不显示 `保存成文本文件`
- 同时有文件和文本时，两个菜单项都显示
- 重名处理正确：`未命名.txt` -> `未命名 2.txt`…
- 复制失败场景（权限不足/目标不可写）不崩溃，日志可见（Console 查看 `RightMenu`）
